Anthropicは3月2日（現地時間2月28日）、Claude Code v2.1.64をリリースした。信頼性の向上、パフォーマンス改善、セキュリティ修正を含むアップデートとなっている。

## Remote Controlサブコマンドの追加

本バージョンの新機能として、`claude remote-control`サブコマンドが追加された。このコマンドにより、外部ビルド向けのローカル環境提供的機能が利用可能になった。CI/CDパイプラインやリモート開発環境での活用が期待される。

## BashToolのパフォーマンス改善

BashToolの改善としてシェルスナップショットが利用可能な場合、デフォルトでログインシェル（-lフラグ）をスキップするようになった。これによりコマンド実行のパフォーマンスが向上し、反復的な開発ワークフローが高速化する。

## コンテキスト自動保存機能の強化

Claudeが自動的に有用なコンテキストをauto-memoryに保存する機能が強化された。セッションをまたいでの作業継続性が向上し、重要な情報が失われるリスクが軽減される。

## Bashコマンド提案の智能化

 compound bashコマンド（例：`cd /tmp && git fetch && git push`）に対する「常に許可」プレフィックス提案が改善された。各サブコマンドごとに より賢明なプレフィックスを計算するようになった。

## マルチエージェントセッションのメモリ最適化

マルチエージェントセッションにおけるメモリ使用量が最適化され、完了したサブエージェントのタスク States が解放されるようになった。長時間稼働する複雑なマルチエージェント構成でのリソース効率が改善されている。

## その他の新機能

- **モデルコマンドの強化**: `/model`コマンドがスラッシュコマンドメニューに現在アクティブなモデルを表示するようになった
- **ツール結果の永続化改善**: 50K文字以上のツール結果はディスクに永続化される（以前は100K）。コンテキストウィンドウの使用量が削減され、会話の longevity が向上
- **Plugin Marketplaceのタイムアウト延長**: デフォルトのgitタイムアウトが30秒から120秒に延長され、`CLAUDE_CODE_PLUGIN_GIT_TIMEOUT_MS`環境変数での設定も可能に
- **npmレジストリのカスタムサポート**: npmソースからのプラグインインストール時にカスタムnpmレジストリと特定バージョン指定が可能に
- **プロジェクト設定のWorktree間共有**: プロジェクト設定とauto-memoryが同一リポジトリのGit Worktree間で共有されるように

## セキュリティ修正

複数のセキュリティ脆弱性が修正されている:

- **statusLine/FileSuggestionフック**: インタラクティブモードでワークスペース信頼の受理なしにコマンドが実行できる問題を修正
- **HTTP Hooks**: ヘッダー値から任意の環境変数をInterpolationできる問題を修正

## バグ修正

- **UIフリッカー**: ユーザー入力が送信後に短く消える問題を修正
- **Windowsクラッシュ**: Windowsでの複数のクラッシュを修正（panic "switch on corrupted value"、多数プロセスspawn時の問題、2分後に発生するWindows ARM64でのクラッシュ）
- **--worktree問題**: 初回起動時に--worktreeが無視される問題を修正
- **MCP OAuthトークンレース**: 複数のClaude Codeインスタンス同時実行時のMCP OAuthトークンリフレッシュレース条件を修正
- **設定ファイル破損**: 複数のClaude Codeインスタンス同時実行時に認証が消失する設定ファイル破損問題を修正
- **リスナーリーク**: ブリッジポリループ、MCP OAuthフロークリーンアップ、インタラクティブパーミッション Handlerでのリスナーリークを修正
- **メモリリーク**: bashコマンドプレフィックスキャッシュ、MCPツール/リソースキャッシュ、JSON解析キャッシュ、gitルート検出キャッシュでのメモリリークを修正
- **ローカルスラッシュコマンド出力**: `/cost`などのローカルスラッシュコマンド出力がユーザーメッセージではなくシステムメッセージとして表示されるように修正
- **重複control_responseメッセージ**: 重複したcontrol_responseメッセージによるAPI 400エラーを修正
