# Claude Code v2.1.70 リリース — 新規Slashコマンドとメモリ機能強化

Claude Code v2.1.70が2026年3月上旬（米国時間）にリリースされた。今回は新規Slashコマンドの追加、メモリ機能の大幅強化、API・ツールの改善など、多くの新機能が実装されている。

## 主な変更点

### 新規Slashコマンドの追加

新たに2つのバンドル済みSlashコマンドが追加された。

- **/simplify**: コードの簡素化を支援するコマンド。複雑なコードをよりシンプルで読みやすい形式にリファクタリングする。
- **/batch**: バッチ処理を実行するためのコマンド。複数のタスクを一度に処理したい場合に便利。

### メモリ機能の強化

Claude Codeのメモリ機能が大幅に強化された。

- **auto-memory**: Claudeが自動的に有用的と判断したコンテキストを「auto-memory」に保存するようになった。ユーザーは`/memory`コマンドで管理可能。
- **マルチエージェントセッションの改善**: マルチエージェントセッションにおけるメモリ使用が改善され、より効率的な情報共有が可能に。

### コード処理の改善

- **/copyコマンド**: コードブロックのインタラクティブなピッカーを提供する新コマンド。コピーしたいコードブロックを選択肢から選べむ。

### Bash統合の改善

 compound bashコマンドに対して、よりスマートなサブコマンドプレフィックスが導入された。これにより、コマンド候補の質が向上し、開発者がより効率的にターミナルを使用できる。

### APIとツールの改善

- **HTTP hooks**: HTTP POSTリクエストを送受信できる機能が追加された。外部サービスとの柔軟な連携が可能に。
- **ENABLE_CLAUDEAI_MCP_SERVERS=false 環境変数**: claude.ai MCPサーバーを無効にするオプションが追加され、プライバシーやセキュリティの要件に応じた設定が可能に。

### モデルコマンドの強化

**/model**コマンドが強化され、現在のアクティブなモデルがSlashコマンドメニュー内に表示されるようになった。

### 安定性とパフォーマンス

多数の問題が修正されている。

- MCP OAuthフローのリスナーリーク修正
- bashコマンドプレフィックスキャッシュのメモリリーク修正
- UIのちらつき修正
- WindowsおよびARM64 Windowsにおけるクラッシュ修正
- その他 various バグ修正

## まとめ

Claude Code v2.1.70は、開発者の生産性向上を目指して多くの新機能が追加されたリリースとなっている。特に`/simplify`や`/batch`といった新しいSlashコマンド、auto-memory機能強化は、日常的な開発ワークフローを大きく改善する可能性を感じる。HTTP hooksの導入により、外部ツールとの連携も更容易になり、今後のエコシステム発展が期待される。

ユーザーは`claude --version`コマンドで 最新バージョンに 更新できる。
