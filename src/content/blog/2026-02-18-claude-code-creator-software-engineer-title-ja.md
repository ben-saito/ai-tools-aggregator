# Claude Codeクリエイター、「ソフトウェアエンジニア」の肩書きは2026年に消えていくと予測。AIがコーディングを「事実上解決した」

AnthropicでClaude Codeを開発したBoris Cherny氏が、Y Combinatorのポッドキャストでのインタビューにおいて、AIがコーディングを「事実上解決した（practically solved coding）」と発言した。同氏はさらに、2026年はAI開発において「とんでもない（insane）」進展がもたらされると予測している。

このインタビューはBusiness Insiderが報じており、2026年2月18日にReddit「r/ClaudeAI」に投稿されたスレッドで101件のコメントを集め、コミュニティで活発な議論を呼んでいる。

## 「ソフトウェアエンジニア」の肩書きが変化する

Cherny氏は、ソフトウェアエンジニアは今後コーディング以外の業務を担うようになっていくと述べ、「ソフトウェアエンジニア」という肩書き自体が2026年を通じて変化していくと予測した。

同氏の発言は、AIがコードを生成する能力が実用レベルに達したという認識に基づいており、Claude Code自体がその象徴的な製品として挙げられている。

## Claude Codeとは

Claude Codeは、Anthropicが開発したターミナルベースのAIコーディングアシスタントである。開発者はコマンドラインから自然言語で指示を与え、コードの生成・修正・レビューをAIと協調して行うことができる。

2026年2月時点で、Claude Codeはすべての有料Claudeプランで利用可能であり、Claude Sonnet 4.6のリリースに合わせてエージェント機能が大幅に強化されている。

## コミュニティの反応

Reddit「r/ClaudeAI」では101件のコメントが寄せられ、ソフトウェアエンジニアの将来的な役割変化について様々な意見が交わされている。

「コーディングを解決した」という発言には賛否両論があり、AIが定型的なコーディング作業を代替する一方で、アーキテクチャ設計、要件定義、品質管理、チームコミュニケーションなどの役割はむしろ重要性を増すという見方も示されている。

## 業界のトレンドとの整合性

Cherny氏の発言は、近年の業界動向と符合している。同時期に、10年以上の経験を持つSalesforce開発者がClaude Opus 4.6を使用して通常1週間かかる作業を1セッションで完了したと報告するなど、AIコーディングツールの実用性が現場で確認されつつある。

Spotifyでは最上級エンジニアが2025年12月以降コードを書かず、AIが生成したコードの監督のみを行っているとCTOが発言するなど、大企業でも開発者の役割変化が具体的に進みつつある。

Cherny氏の「ソフトウェアエンジニア」という肩書きの変化という予測は、コーディングという技術的作業がAIによって代替される一方で、エンジニアリングの本質的な役割（設計、判断、品質管理）は継続するという見方を示している。
