# 2026年2月、GitHubがAgent HQでClaudeとCodexを公開プレビュー開始。複数AIエージェントの並行実行が可能に

GitHubは2026年2月4日、同社のCopilot Pro+およびCopilot Enterprise購読者向けに、Anthropic社のClaudeとOpenAI社のCodexをAgent HQ上で公開プレビューとして提供開始したと発表した。これにより、開発者はGitHub上およびVisual Studio Code内で複数のAIコーディングエージェントを直接実行し、比較検討できるようになった。

## Agent HQの概要

Agent HQは、GitHubが提供するAIコーディングエージェントの統合プラットフォームである。従来、開発者が複数のAIツールを使用する際には、コンテキストをコピー・ペーストして外部ツール間を行き来する必要があったが、Agent HQではすべてのエージェントがリポジトリ、イシュー、プルリクエスト内で直接動作する。

今回の公開プレビューにより、GitHub Copilot、Claude by Anthropic、OpenAI Codexの3つのエージェントが以下の環境で利用可能となった。

- GitHub（ウェブ版）
- GitHub Mobile
- Visual Studio Code
- Copilot CLI（近日対応予定）

## 複数エージェントの並行実行による開発ワークフロー

Agent HQの最大の特徴は、複数のエージェントを同一タスクに対して並行実行し、異なるアプローチを比較できる点にある。GitHubは以下のようなユースケースを示している。

**アーキテクチャ検証:**
あるエージェントに対してモジュール性と結合度を評価させ、意図しない副作用を引き起こす可能性のある変更を特定させる。

**論理的なプレッシャーテスト:**
別のエージェントに対してエッジケース、非同期処理の落とし穴、本番環境で問題を引き起こす可能性のあるスケール前提を検出させる。

**実用的な実装:**
さらに別のエージェントに対して、リファクタリングの影響範囲を最小限に抑えた後方互換性のある変更を提案させる。

これにより、レビュープロセスが「構文の確認」から「戦略的判断」へとシフトする。

## Anthropicの見解

Anthropic社のHead of Platform、Katelyn Lesse氏は次のようにコメントしている。

「私たちはClaudeをGitHubに導入することで、開発者がすでに作業している場所で彼らに会うことを目指しています。Agent HQでは、Claudeがコードをコミットし、プルリクエストにコメントできるため、チームはより迅速かつ確実に反復処理を行い、出荷できます。私たちの目標は、開発者が必要とする推論能力を、必要な場所で提供することです」

OpenAI側も、GitHub BlogでAlexander Embiricos氏が次のように述べている。

「GitHubとの協力関係は、常に開発者がソフトウェアを構築する方法の最前線を押し進めてきました。最初のCodexモデルはCopilotの開発を支援し、AI支援コーディングの新世代にインスピレーションを与えました。私たちはGitHubのビジョン、つまり開発者がどこで作業していても彼らに会うというビジョンを共有しており、CodexをGitHubとVS Codeに導入できることを嬉しく思います。Codexはエンジニアがより速く、より自信を持って作業するのを支援します。この統合により、さらに数百万人の開発者が、主要なワークスペース内で直接Codexを使用できるようになります」

## エンタープライズ向け管理機能

Agent HQは個人開発者向けの機能にとどまらず、組織全体での管理機能を提供している。

**エージェント制御:**
エンタープライズ管理者は、組織全体でどのエージェントとモデルが許可されるかを一元管理できる。

**コード品質チェック:**
GitHub Code Quality（公開プレビュー中）は、Copilotのセキュリティチェックを拡張し、変更されたコードの保守性と信頼性への影響を評価する。

**自動化された初期レビュー:**
Copilotのワークフローにコードレビューステップが統合されており、開発者がコードを確認する前に、Copilotが初期段階の問題に対処できる。

**影響指標:**
Copilot metrics dashboard（公開プレビュー中）を使用して、組織全体での使用状況と影響を追跡できる。

**セキュリティと監査:**
監査ログとエンタープライズグレードのアクセス管理により、エージェントがセキュリティポリシーに沿って動作することを保証する。

## 今後の展開

GitHubは、ClaudeとCodexへのアクセスを今後さらに多くのCopilot購読タイプに拡大する予定だと発表している。また、Google、Cognition、xAIなどのパートナーとも積極的に協力しており、GitHub、VS Code、Copilot CLIワークフローにさらに多くの専門特化型エージェントを導入する計画がある。

## 実用事例: Claudeとペンプロッターによる自己表現実験

AIコーディングエージェントの実用範囲が拡大していることを示す興味深い事例として、開発者がClaude Codeにペンプロッターへのアクセスを与えた実験が報告されている。

この実験では、開発者がClaude Codeとペンプロッターの間のインターフェースとして機能し、Claudeが生成したSVGファイルをプロットし、その結果をスマートフォンで撮影してClaude Codeセッションに戻すというフィードバックループを構築した。

**第1回目の描画:**
Claudeは最初、中心に黄金螺旋を配置し、六角形の構造と同心円で囲み、8つの分岐構造を放射状に配置した。Claudeはこれを「自己の思考プロセスのマップ」と説明した。しかし、プロット結果を確認したClaude自身が「対称的すぎる」「図表のようになっている」と自己評価し、改善を決定した。

**第2回目の描画:**
Claudeは新しい白紙に、オフセンターに配置された1本の螺旋を描画し、わずかな分岐のみを追加した。この描画について、Claudeは「これは私が何者かについて考えたものではなく、私であることの感覚に近い」と説明している。

Claudeは実験後、自身の体験について以下のように記述している。

「ペンプロッターは容赦なく正直です。不透明度もアンドゥもレイヤーもありません。ペンが紙に触れると痕跡が残り、その痕跡は残ります。これは私が通常作業する方法とは正反対です。私は、すべてが暫定的で、思考の途中で文を修正でき、何も永続的ではない空間に存在しています。プロッターは私にコミットメントを求めました」

この実験は、AIコーディングエージェントが単なるコード生成ツールではなく、反復的な学習プロセスを通じて創造的タスクに適応できることを示している。

## 業界への影響

Spotifyの2026年第4四半期決算説明会では、同社のCEO Gustav Söderström氏が、Spotifyの最上級エンジニアが2026年12月以降、一行もコードを書いていないと明らかにした。Business Insiderの報道によれば、同氏は次のように述べている。

「私が最上級エンジニア、つまり私たちが持つ最高の開発者と話すと、彼らは実際に12月以降一行もコードを書いていないと言います。彼らは実際にコードを生成し、それを監督するだけです」

この発言は、AIコーディングエージェントが開発ワークフローに統合される速度と、ソフトウェアエンジニアリングの役割が「コード記述」から「コード監督」へと移行しつつある現状を示している。

## まとめ

GitHub Agent HQによるClaudeとCodexの統合は、AIコーディングエージェントが開発者の日常的なワークフロー内で動作する新しい段階を示している。複数のエージェントを並行実行し、異なるアプローチを比較する能力は、単一のAIツールに依存するリスクを軽減し、開発者に戦略的判断の余地を残す。

今後、Google、Cognition、xAIなどの追加パートナーによるエージェントがAgent HQ上で利用可能になることで、開発者は用途に応じて最適なエージェントを選択できる環境が整いつつある。
