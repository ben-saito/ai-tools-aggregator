# Workflow Optimization - ログインで止まらない仕組み

## 🎯 目的

**課題**: 毎回ログインで作業が止まる  
**目標**: ログイン不要で自律的に作業を進められる仕組み構築

---

## 💡 解決策（優先順位順）

### Solution 1: Chrome拡張（Browser Relay）- 最優先

#### 概要
つとむ様が既にログインしているブラウザセッションに接続し、そのセッションを使って操作する。

#### メリット
- ✅ ログイン不要（つとむ様のセッション活用）
- ✅ Google AI Studio、Twitter、Product Huntなど全対応
- ✅ リアルタイムで画面確認可能
- ✅ 一度接続すれば継続使用可能

#### 設定方法

**Step 1: Chrome拡張の確認**
- Clawdbot Browser Relayが既にインストールされているか確認
- 未インストールなら追加

**Step 2: 使用手順**
1. つとむ様が目的のサイト（例: Google AI Studio）を開く
2. Chrome拡張のアイコンをクリック
3. 「接続」状態になる（バッジが表示される）
4. 私がそのタブを操作可能になる
5. 作業完了後、接続解除

**Step 3: 具体的なコマンド**
```
私: browser action:open targetUrl:https://aistudio.google.com profile:chrome
（つとむ様のChromeタブに接続）
```

#### 適用例
- Google AI Studio（画像生成）
- Twitter（アカウント作成、投稿）
- Product Hunt（アカウント作成、Launch準備）
- Canva（ロゴ作成）
- その他ログイン必要なサービス全般

---

### Solution 2: APIキーの事前設定

#### 対象サービス

**1. Gemini API（画像生成）**
- 現状: 無料枠のAPIキー使用（クォータ制限あり）
- 改善: 有料プランのAPIキー設定（必要に応じて）
- 用途: プログラマティックな画像生成

**2. Twitter API（将来）**
- 用途: ツイート自動投稿、フォロワー管理
- 設定: Twitter Developer Accountで取得
- 優先度: 中（手動投稿で当面OK）

**3. Product Hunt API（将来）**
- 用途: Launch自動化、データ取得
- 優先度: 低（Launch時に検討）

#### 設定手順

**Gemini API（画像生成）**
```bash
# 環境変数に設定
export GEMINI_API_KEY="your-api-key-here"

# またはClawdbot設定ファイルに追加
~/.clawdbot/clawdbot.json
{
  "skills": {
    "nano-banana-pro": {
      "env": {
        "GEMINI_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

**確認方法**
```bash
# APIキーが設定されているか確認
echo $GEMINI_API_KEY

# または
cat ~/.clawdbot/clawdbot.json | grep GEMINI_API_KEY
```

### Solution 3: ログイン不要ツールの優先使用

#### 戦略
ログインが必要なツールの代わりに、ログイン不要またはAPI経由で使えるツールを優先的に使用。

#### 具体例

**画像生成**
- ❌ Google AI Studio（Webログイン必要）
- ✅ Gemini API（APIキー設定後は自動）
- ✅ 暫定ロゴ（絵文字 🤖）→ ログイン不要
- ✅ Bing Image Creator（APIまたはWeb、後者はログイン必要）

**ロゴ作成**
- ❌ Canva（ログイン必要）
- ✅ 暫定ロゴ（絵文字）→ 即使用可能
- ✅ Gemini API → 設定後は自動

**SNS投稿**
- ❌ Twitter Web（ログイン必要）
- ✅ Twitter API（将来、APIキー設定後）
- ⚠️ 手動投稿（当面はこれで十分）

#### 判断基準

**ログイン必要なツールを使う場合**:
1. 代替手段がない
2. Chrome拡張で接続可能
3. つとむ様が既にログインしている

**ログイン不要なツールを使う場合**:
1. 代替手段がある
2. API経由で使える
3. すぐ実行できる

## 🔧 実装ガイド

### Chrome拡張の使い方

#### 接続方法
```
1. つとむ様: 目的のサイトを開く
2. つとむ様: Chrome拡張アイコンをクリック
3. 私: browser action:snapshot profile:chrome で確認
4. 私: browser action:act で操作
```

#### 切断方法
```
1. 私: browser action:close で完了
2. つとむ様: Chrome拡張アイコンをクリック（切断）
```

#### 注意点
- つとむ様のブラウザで操作が見える
- セキュアな情報は慎重に扱う
- 作業完了後は必ず切断
