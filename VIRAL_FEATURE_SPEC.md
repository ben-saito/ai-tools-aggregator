## 2. 機能仕様

### 2.1 診断フロー

#### Step 1: ランディングページ（/quiz）
- **見出し:** 「あなたに最適なAIツールを見つけよう」
- **サブタイトル:** 「5つの質問で、あなたのニーズに合った最適なAIツールを診断します」
- **CTA:** 「診断を始める」ボタン
- **所要時間表示:** 「約1分で完了」
- **過去の診断数表示:** 「これまで12,345人が診断しました」（バイラル促進）

#### Step 2-6: 質問フロー
**プログレスバー表示:** 「5問中3問目」

**質問1: 主な目的は？**
- [ ] 仕事の効率化（業務自動化、ドキュメント作成）
- [ ] 創作活動（デザイン、動画、音楽制作）
- [ ] 学習・スキルアップ（プログラミング、語学、知識習得）
- [ ] ビジネス成長（マーケティング、営業、データ分析）

**質問2: 主な用途は？**
- [ ] テキスト作成・編集（ブログ、メール、レポート）
- [ ] 画像・動画生成（デザイン、イラスト、動画編集）
- [ ] コード開発（プログラミング、デバッグ、コードレビュー）
- [ ] データ分析・リサーチ（調査、分析、レポート作成）
- [ ] その他（音声、音楽、翻訳、SEO等）

**質問3: 予算は？**
- [ ] 無料のみ（コストをかけたくない）
- [ ] 月1,000円まで（お試し程度）
- [ ] 月5,000円まで（業務効率化に投資OK）
- [ ] 予算は気にしない（最高のツールを使いたい）

**質問4: スキルレベルは？**
- [ ] 初心者（AIツールを初めて使う）
- [ ] 中級者（いくつか使ったことがある）
- [ ] 上級者（複数のツールを使いこなしている）
- [ ] プロフェッショナル（業務で日常的に使用）

**質問5: 利用形態は？**
- [ ] 個人利用（自分だけで使う）
- [ ] 小規模チーム（2-10人）
- [ ] 中規模チーム（11-50人）
- [ ] 企業利用（50人以上、エンタープライズ向け）

#### Step 7: 診断結果ページ（/quiz/result/[id]）

**結果表示:**
- 「あなたにおすすめのAIツール」見出し
- おすすめツール3-5個をカード形式で表示
- 各カードの内容:
  - ツール名
  - アイコン/ロゴ
  - 一行説明
  - マッチ度（85%、92%等）
  - 料金（無料、$10/月等）
  - 「詳細を見る」ボタン
  
**診断結果サマリー:**
- 「あなたのプロフィール:」
  - 目的: 仕事の効率化
  - 用途: テキスト作成・編集
  - 予算: 月5,000円まで
  - スキル: 中級者
  - 利用形態: 個人利用

**アクション:**
- 🔄 「もう一度診断する」ボタン
- 📤 「結果をシェア」ボタン（Twitter、Facebook、LinkedIn）
- 📧 「結果をメールで送る」ボタン

**OGP画像:**
- 診断結果を画像化（Twitter等でシェア時に表示）
- 内容: おすすめツール名、マッチ度、プロフィール概要

### 2.2 診断ロジック（スコアリングアルゴリズム）

#### マッチングルール

**Step 1: カテゴリーフィルタリング**
- 質問1（目的）+ 質問2（用途）でカテゴリー絞り込み
- 例: 「仕事の効率化 + テキスト作成」→ Text Generation カテゴリー

**Step 2: 予算フィルタリング**
- 質問3（予算）で価格帯フィルタリング
- 無料のみ → Free のみ
- 月1,000円まで → Free + Freemium（$10/月以下）
- 月5,000円まで → Free + Freemium + Paid（$50/月以下）
- 予算気にしない → すべて

**Step 3: スキルレベルマッチング**
- 質問4（スキルレベル）でツールの難易度マッチング
- 初心者 → 使いやすさ重視（UI/UX評価高いツール）
- 中級者 → バランス重視
- 上級者 → 機能豊富なツール
- プロフェッショナル → エンタープライズ機能重視

**Step 4: 利用形態マッチング**
- 質問5（利用形態）でチーム機能の有無
- 個人利用 → チーム機能不要
- チーム利用 → コラボレーション機能重視

**Step 5: スコアリング**
各ツールに対してマッチ度スコア（0-100%）を計算:
```
スコア = (カテゴリー一致 × 40%) + (価格適合 × 20%) + (スキルマッチ × 20%) + (利用形態マッチ × 10%) + (人気度 × 10%)
```

**Step 6: ランキング**
- スコア上位3-5個を推薦
- 同スコアの場合、人気度（ランキング順位）で優先順位付け

## 3. 技術仕様

### 3.1 フロントエンド

**技術スタック:**
- Astro（既存プロジェクト）
- React（質問フロー用）
- Tailwind CSS（スタイリング）

**ページ構成:**
```
src/pages/
├── quiz/
│   ├── index.astro          # ランディングページ
│   ├── [step].astro         # 質問ステップ（動的ルート）
│   └── result/
│       └── [id].astro       # 結果ページ
src/components/
├── quiz/
│   ├── QuizLanding.astro    # ランディングコンポーネント
│   ├── QuizStep.tsx         # 質問ステップ（React）
│   ├── QuizProgress.tsx     # プログレスバー
│   ├── QuizResult.astro     # 結果表示
│   ├── ToolCard.astro       # ツールカード
│   └── ShareButtons.tsx     # シェアボタン
```

**状態管理:**
- LocalStorage（質問の回答を保存、ブラウザバック対応）
- URL パラメータ（結果ページの共有用）

### 3.2 バックエンド

**API エンドポイント:**

**POST /api/quiz/submit**
- リクエスト: 質問の回答（JSON）
```json
{
  "answers": {
    "purpose": "work-efficiency",
    "usage": "text-generation",
    "budget": "5000",
    "skill": "intermediate",
    "team": "individual"
  }
}
```
- レスポンス: 診断結果ID + おすすめツール
```json
{
  "resultId": "abc123def456",
  "recommendations": [
    {
      "toolId": "chatgpt",
      "matchScore": 92,
      "tool": { ... }
    },
    ...
  ]
}
```

**GET /api/quiz/result/[id]**
- リクエスト: 結果ID
- レスポンス: 診断結果データ

**データベーススキーマ:**
```sql
CREATE TABLE quiz_results (
  id VARCHAR(255) PRIMARY KEY,
  answers JSONB NOT NULL,
  recommendations JSONB NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  ip_address VARCHAR(45),
  user_agent TEXT
);

CREATE INDEX idx_quiz_results_created_at ON quiz_results(created_at);
```

## 5. バイラル戦略

### 5.1 バイラル要素

**1. ソーシャルプルーフ（社会的証明）**
- 「これまで12,345人が診断しました」
- 「上位3%のマッチ度です！」（結果に表示）
- ツイート時の表示「#AITools診断 で1万人が診断！」

**2. パーソナライゼーション**
- 結果が個人ごとに異なる → 「私の結果は？」という好奇心
- マッチ度スコア → 「92%マッチ！」と自慢したくなる

**3. シェアしやすさ**
- ワンクリックでTwitterシェア
- OGP画像で結果がビジュアル化 → Twitter上で目立つ
- シェアテキストが既に書かれている（編集可能）

**4. 比較したくなる仕組み**
- 「友達と比較してみよう！」CTA
- 「あなたは〇〇タイプ！」（パーソナリティ診断風）

### 5.2 拡散チャネル

**1. Twitter（最重要）**
- シェアボタン
- ハッシュタグ: #AITools診断 #AI #ProductHunt
- メンション促進: 「@ai_tools_hub で診断結果をシェアしてみよう！」

**2. Product Hunt**
- Launch時のコメント欄で「診断してみてください！」
- Maker（開発者）としてコメント返信時に診断を勧める

**3. Reddit**
- r/artificial、r/ChatGPT、r/ProductivityApps等
- 「面白い診断ツール作りました」投稿

**4. LinkedIn**
- ビジネス系AIツールユーザーに訴求
- 「仕事効率化のためのAIツール診断」

**5. Hacker News**
- 技術的な側面を強調
- 「診断ロジックのアルゴリズム」記事も作成

### 5.3 初期ブースト戦略（Launch Day）

**事前準備（Launch前1週間）:**
- メールリスト作成（既存訪問者、Product Hunt フォロワー）
- AIインフルエンサーに事前DM（10-20人）
- Discord/Slackコミュニティに告知

**Launch Day（00:01 PST / 17:01 JST）:**
- メールリスト全員に「診断してみて！」メール送信
- Twitter/X で連続投稿（1時間ごと）
- Product Huntコメント欄で診断を勧める
- 個別DM送信（AIインフルエンサー、友人、過去のサポーター）

**目標数値:**
- 初日診断数: 5,000回
- Twitterシェア: 1,000回
- Product Hunt upvote: 300票

## 9. リスクと対策

### 9.1 技術的リスク

**リスク1: OGP画像生成が遅い**
- 対策: Vercel Edge Functions使用、画像キャッシュ

**リスク2: データベース負荷（初日5,000診断）**
- 対策: Supabase無料枠で対応可能（月500MB、50,000行まで）

**リスク3: APIレート制限**
- 対策: キャッシュ戦略、CDN使用

### 9.2 マーケティングリスク

**リスク1: Product Hunt 1位取れない**
- 対策: トップ10入りでも十分な露出（1日2-5万PV）

**リスク2: シェア率が低い**
- 対策: シェアボタンを目立たせる、シェア特典（ツール追加情報等）

**リスク3: 診断精度が低い**
- 対策: 初期ユーザーのフィードバックで改善、A/Bテスト

### 9.3 競合リスク

**リスク: 似た診断ツールが既にある**
- 対策: 差別化（最新AIツール、日本語対応、ランキングデータ連動）
